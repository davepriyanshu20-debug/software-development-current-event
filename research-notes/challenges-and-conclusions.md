# Challenges, Limitations & Conclusion — GitHub Copilot

## 1. Code Accuracy & Complexity Issues
- Copilot still struggles with context window limits and can produce errors during complex, multi-file refactoring
- Generated code often needs significant rework before it's production-ready — it's a starting point, not a finished product

## 2. Privacy & Copyright Concerns
- Enterprise safeguards exist, but teams still need to understand how their data is handled
- Suggested code can resemble snippets from public repositories, raising copyright/licensing questions that developers need to verify

## 3. Over-Reliance & Skill Erosion
- Relying on AI for complex logic can let incorrect or insecure code slip through without careful review
- Ongoing industry debate: when AI suggestions fail, is it a tool limitation or a "prompt engineering" problem? This points to real limits in current AI reasoning

## Conclusion Script (~30 sec)
"So is GitHub Copilot changing software development for the better? The honest answer is: it depends on how it's used. It speeds up repetitive work and lowers the barrier to entry, but it isn't a replacement for understanding your own code — developers still need to review, test, and verify everything it suggests. As AI coding tools keep evolving, the real skill isn't writing code from scratch anymore — it's knowing when to trust the AI, and when to double-check it."
